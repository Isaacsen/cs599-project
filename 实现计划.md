# Software Engineer Agent 实现计划

当前项目定位为 **Software Engineer Agent + 权限隔离执行平台**，主入口是 `src.engineer` 的 LangGraph StateGraph 工作流。早期的 `src.main`、Benchmark 和独立 Agent CLI 保留为辅助演示、离线兜底和评估工件。

## 一、已完成主流程

推荐验收命令：

```bash
python -m src.engineer examples/review_target --use-llm-review --use-llm-tests --run-sandbox --sandbox-executor docker --docker-image software-engineer-agent-python:latest --output docs/runs/software_engineer.json --output-md docs/runs/software_engineer.md
```

当前完整 Agent 流程：

```text
Repo Scanner
  -> Rule Code Reviewer Agent
  -> LLM Code Reviewer Agent
  -> Bug Fixer Agent
  -> Patch Reviewer Agent
  -> Unit Test Writer Agent
  -> LLM Test Generator Agent
  -> Sandbox Validator Agent
  -> Repair Loop Agent
  -> Coverage Feedback Agent
  -> JSON / Markdown Report
```

已完成能力：

1. SDD 规格文档：`product_spec.md`、`architecture_spec.md`、`api_spec.md`。
2. LangGraph 状态图：`src/workflow/software_engineer_graph.py`。
3. 规则代码审查：危险调用、硬编码密钥、宽泛异常、缺失测试覆盖、除零风险。
4. 真实 LLM 代码审查：通过 OpenAI-compatible client 接入 DashScope / DeepSeek 等 provider。
5. 自动修 Bug 计划：默认 dry-run，显式 `--apply-fixes` 才写回。
6. Patch 审查：对修复计划做二次安全门禁。
7. 缺失覆盖单测生成：默认 dry-run，显式 `--apply-tests` 才写入。
8. LLM 测试生成：默认使用真实 provider 配置，生成结果经过 Security Checker。
9. 沙箱验证：临时工作区 + Docker 隔离执行 pytest。
10. 修复循环规划：根据 Patch 审查与沙箱结果给出下一步动作。
11. 覆盖反馈：汇总已覆盖和缺失函数。
12. 可观测输出：JSON 报告、Markdown 报告、节点轨迹和运行状态。

## 三、权限隔离实现

已完成的隔离策略：

- Docker `--network none` 禁用网络。
- 源码只读挂载。
- 容器根文件系统只读。
- `/tmp` 使用有限 tmpfs。
- CPU、内存、PID 和 timeout 限制。
- `--cap-drop ALL` 与 `no-new-privileges`。
- 生成代码执行前先经过 Security Checker。
- 写回源码或测试文件必须显式开启 apply 开关。

## 四、最终验证计划

推荐使用：

```powershell

```

1. `python -m unittest discover -s tests`

3. 构建 `software-engineer-agent-python:latest` Docker 沙箱镜像。
4. 运行完整 `src.engineer` Agent。
5. 运行真实 LLM 测试生成 Agent。
6. 导出并检查最终 PDF。
7. 扫描明文 API Key。
